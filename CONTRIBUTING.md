# Contributing to DayniteJs

First off, thank you for considering contributing to **DayniteJs**! It's people like you that make DayniteJs such a great tool for the community.

## Where do I go from here?

If you've noticed a bug or have a question, [search the issue tracker](https://github.com/suhaibmuhd01/daynitejs/issues) to see if someone else in the community has already created a ticket. If not, go ahead and [make one](https://github.com/suhaibmuhd01/daynitejs/issues/new/choose)!

## Fork & create a branch

If this is something you think you can fix, then [fork DayniteJs](https://github.com/suhaibmuhd01/daynitejs/fork) and create a branch with a descriptive name.

## Local Development Setup

To build and run the project locally, run the following steps:

1. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/daynitejs.git
   cd daynitejs
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Build the library:
   ```bash
   npm run build
   ```

4. Run the demo locally:
   ```bash
   npm run serve
   ```
   Open `http://localhost:3000/demo/` in your browser.

## Running Tests

Before submitting a pull request, ensure that the tests pass:

```bash
npm run test
```

## Creating a Pull Request

When you are ready to generate a pull request, please ensure the following:

- You have tested your code directly in the provided `demo` directory, or wrote a unit test.
- Your PR matches our provided template.
- Your code is clean and linted.

Thank you again for your time and contribution!
